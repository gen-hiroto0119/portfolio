## 背景と課題

バックオフィス部門では、稟議・購買・契約確認がそれぞれ別スプレッドシートで管理されていました。担当者の記憶とチャット履歴に依存する部分が大きく、承認漏れや差し戻しの原因特定に時間がかかっていました。

監査対応時には、過去案件の証跡を手作業で集める必要があり、四半期ごとに相当な工数が発生していました。

## 意思決定

最初から全業務を置き換えるのではなく、最もボトルネックだった購買承認から着手しました。状態遷移を明示したワークフローUIを設計し、各ステップで必要な入力項目を段階的に提示する方式を採用しました。

権限設計はロールベースに加え、金額閾値で承認者が自動切り替わるルールをアプリ側に実装し、運用ルールのブレを減らしています。

## 技術選定

Next.js App Router と Server Actions を使い、フォーム送信から承認更新までを型安全に実装しました。履歴はイベントソーシングに近い形で保存し、後から任意時点の状態を再現できるようにしています。

UIは社内利用を前提に情報密度を高めつつ、1画面1タスクの原則を守り、教育コストを抑えました。

- 申請フォーム: 入力補助とバリデーションをリアルタイム表示
- 承認キュー: 期限順ソートと一括承認（条件付き）
- 監査ビュー: 変更履歴の差分表示とCSVエクスポート

## 成果

購買承認の平均リードタイムは3.2日から1.1日へ短縮され、差し戻し率も低下しました。監査資料はワンクリックで期間指定エクスポートが可能になり、四半期作業の工数は約70%削減されています。

他部門からの横展開要望もあり、現在は契約確認フローの移行を進めています。
